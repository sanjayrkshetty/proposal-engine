# Discovery Brief — Gulf Digital Holdings
**Service:** Web Application Penetration Test  
**Recommended Tier:** Enterprise — The client's requirement for PCI DSS certification and the complexity of their web applications warrant a comprehensive and thorough web application penetration test, which is typically offered in the Enterprise tier.

## Pain Points
- Exposure to financial data breaches
- Insufficient security controls for web applications
- Difficulty in meeting PCI DSS certification requirements
- Vulnerability to common web attacks like SQL injection and cross-site scripting

## Regulatory Drivers
- PCI DSS
- GDPR
- Local financial services regulations in the MEE region

## Environment Assumptions
- Web applications are custom-built and hosted on-premises or in the cloud
- Multiple APIs are integrated with the web applications
- User authentication and authorization mechanisms are in place

## Top Discovery Questions
1. What are the most critical web applications that require testing for PCI DSS certification?
2. What is the current security posture of the web applications and APIs?
3. Are there any existing security vulnerabilities or incidents that need to be addressed?
4. What are the expected outcomes and deliverables from the web application penetration test?
5. Are there any specific compliance or regulatory requirements that need to be met?

## Urgency Signals
- Upcoming PCI DSS certification audit
- Recent security incidents or breaches in the financial services industry
- Increased risk of cyber attacks due to expanding online presence
